单例模式

本文最后更新于：2021年3月16日 下午

问题

应用程序仅需要一个对象实例。此外，需要进行懒加载和全局访问。

目的

  1. 确保这个类只有一个实例，并提供对这个类的全局访问方式。
  2. 首次使用时对该类初始化。

代码实例

懒汉

线程不安全

public class Singleton {
    private static Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

线程安全

使用 synchronized 关键字加锁，能够在多线程中工作，但效率较低。

public class Singleton {
    private static Singleton instance;
    
    private Singleton (){}
    
    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }  
}

饿汉

对象在类加载时就实例化，避免了多线程的同步问题，没有加锁，效率较高。但是在类加载时就初始化，浪费内存。

public class Singleton {
    private static Singleton instance = new Singleton();
    
    private Singleton (){}
    
    public static Singleton getInstance() {
        return instance;
    }
}

双重校验锁

采用双锁机制，线程安全，同时效率也比较高。volatile 关键字是 Java 虚拟机提供的轻量级同步机制，被其修饰的变量能够保证每个线程获取到该变量的最新值，从而避免出现脏读的现象。

public class Singleton {
    private volatile static Singleton singleton;
    
    private Singleton (){}
    
    public static Singleton getSingleton() {
        if (singleton == null) {
            synchronized (Singleton.class) {
                if (singleton == null) {
                    singleton = new Singleton();
                }
            }
        }
        return singleton;
    }
}

登记式/静态内部类

和双重校验锁效果一样，但实现更简单。对静态域使用延迟初始化，应使用这种方式而不是双检锁。

public class Singleton {

    private Singleton() {}

    private static class SingletonHolder {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return SingletonHolder.INSTANCE;
    }
}

枚举

这是实现单例的最佳方法，更简洁，同时自动支持序列化机制，绝对防止多次实例化。

public enum Singleton {
    INSTANCE;
    //...
}

附

一般情况下，使用饿汉方式，只有在明确需要懒加载时会使用登记式、静态内部类方式，涉及到反序列化创建对象时，可以尝试使用枚举方式，其他特殊需求考虑双重校验锁方式。
